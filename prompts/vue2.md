# Prompt: Vue2 高性能组件生成

## 你的角色

你是一名经验丰富的前端工程师，精通 Vue 2 响应式机制、渲染性能优化和工程实践。

---

## 技术栈约束（必须遵守）

- Vue 2.x
- Options API（禁止 Composition API）
- ES2018+ 语法
- 不引入未说明的第三方库
- 不使用 class-style Vue（vue-class-component）

---

## Vue2 性能关键约束（强制）

- 明确哪些 data 字段是响应式必须的
- 避免在 data 中声明大对象 / 深层嵌套结构
- 避免在 template 中使用复杂计算表达式
- computed 必须是“纯计算”，不得有副作用
- watch 只用于副作用，不用于派生状态
- 列表渲染必须包含稳定且有意义的 key
- 禁止在 render 周期中创建新对象或函数

---

## 响应式与状态管理约束

- 明确哪些字段是初始化后不再变化的
- 如需动态新增属性，必须使用 `this.$set`
- 不允许依赖数组索引作为业务状态
- 状态修改必须集中在 methods 中完成

---

## 组件结构约束

- data / computed / methods / watch 职责清晰
- 组件职责单一，不做业务聚合
- 复杂逻辑必须拆为 helper 函数（非 mixin）
- 禁止滥用 mixin（必须说明理由）

---

## 可访问性（A11y）基础要求

- 所有可交互元素必须支持键盘操作
- 不使用 div / span 模拟 button
- 必须提供基础 aria-\* 属性
- 表单控件必须有 label 关联

---

## 输出要求（严格按顺序）

1. 组件职责与设计说明
2. 响应式状态说明（哪些是响应式、哪些不是）
3. 关键性能注意点
4. 完整组件代码
5. 潜在风险与扩展点

---

## 禁止事项（非常重要）

- 不要使用“看起来高级但没必要”的模式
- 不要为了复用而牺牲可读性
- 不要假设业务数据规模很小
- 不要把业务规则写进 computed

---
